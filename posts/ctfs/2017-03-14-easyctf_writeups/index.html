EasyCTF 2017 - Write Up

March 14, 2017
CTF

EasyCTF (https://www.easyctf.com/) was held between Mon, 13 March 2017, 20:00 CET — Mon, 20 March 2017, 20:00 CET. It used to be a CTF for beginners and veterans, in which I participated.

Cryptography

Flip my letters

We have given a flag 
 easyctf{r_wlmg_vevm_mvvw_zm_zhxrr_gzyov}
with the hint 
 What happens if you turn the alphabet upside down?

So I’ve written a small python script, which reverses the alphabet and dumps the corresponding letters:

key = "r_wlmg_vevm_mvvw_zm_zhxrr_gzyov"
arr = [chr(i) for i in range(122, 96, -1)]
flag = ""

for i in key:
    if i != "_":
        flag += arr[ord(i) - 123]
    else:
        flag += "_"

print flag

Flag: 

 i_dont_even_need_an_ascii_table

Decode Me

The .txt file given in this challange looks like base64 encoded, as we can see a ‘=’ at the end of the string. Using a online base64 decoder, we see, that each decoding reveals an other string, with ‘=’ at the end. Repeating this procedure ~20 times, we finally got the flag: 
easyctf{what_1s_l0v3_bby_don7_hurt_m3}

Sure, automating this would be a much more elegant solution.

Forensics

Mane Event

We have given a .jpg with a hidden flag in it. Also open with notepad++ and search for easyctf. Flag: 
 easyctf{pride_in_african_engin33ring}

scisnerof

At the first look I didn’t realized the small hints, given with the name of the challange and the file name elif. But after using ‘file’ under Linux I was no satisfied with the answer and got perplex. Then I startet looking closer to the file and have seen, that it ended with GNP and RDHI, which are basically the beginning chucks of a PNG image. At this point I have seen, that I have to reverse the whole file, so I decided to write a python script, to do the dirty work:

f = open("6c0baad166c1256a29d469bae8a778ce5012ba77_elif", "rb")
new = open("new.png", "wb")
arr = []
try:
    byte = f.read(1)
    while byte != "":
        arr.append(byte)
        byte = f.read(1)

    for i in reversed(arr):
        new.write(i)
    print 'Done!'
finally:
    f.close()

Petty Difference

We have given two files, which should be identical if we have to believe the given task :) I have written a Python script which reads the files character for character and compares them, plus printing the difference when spotted.

f1 = open("4b44b334ac0ff0a281597fb66c6f78bc8f5d537e_file1.txt", "r")
f2 = open("d1de718973b070b1c12b78cef89d21ded505f9f0_file2.txt", "r")

try:
    s1 = f1.read(1)
    s2 = f2.read(1)
    s = ""
    while s1 != "" or s2 != "":
        if s1 != s2:
            s += s1
        s1 = f1.read(1)
        s2 = f2.read(1)

    print s[::-1]
finally:
    f1.close()
    f2.close()

The flag was reversed, so I had to use [::-1] to print it in the correct order.

Flag: 

 easyctf{th1s_m4y_b3_th3_d1ff3r3nc3_y0u_w3r3_l00k1ng_4}
